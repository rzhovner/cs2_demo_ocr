# Ground Truth Data

Engagement CSVs generated by `cs2_engagement_parser.py` from `.dem` demo files. These serve as the canonical reference for validating all video/audio-based detectors.

## How to Generate

```bash
python src/cs2_engagement_parser.py data/your_demo.dem --output tests/ground_truth/your_demo_engagements.csv
```

## Expected Files

| File | Source Demo | Session Type | Engagements |
|---|---|---|---|
| `training1_engagements.csv` | `training1.dem` | Tapping | ~100 |
| `training2_spraying_engagements.csv` | `training2_spraying.dem` | Spraying | ~145 |

## Schema

Each row is one engagement (one bot killed). Key columns:

| Column | Type | Description |
|---|---|---|
| `engagement_id` | int | Sequential 1, 2, 3... |
| `kill_time_s` | float | Kill timestamp (seconds from demo start) |
| `engagement_start_s` | float | Start of engagement (previous kill time) |
| `ttff_ms` | float | Time to first fire (ms) |
| `ttk_ms` | float | Time to kill (ms) |
| `shots_fired` | int | Total shots in this engagement |
| `shots_hit` | int | Shots that matched a hurt event |
| `hit_rate` | float | shots_hit / shots_fired |
| `headshot` | bool | Whether killing blow was headshot |
| `hit_sequence` | str | Per-shot pattern, e.g. `M,Y,Y` |

## Note on .gitignore

The root `.gitignore` has `*_engagements.csv` to ignore regeneratable parser output. An exception `!tests/ground_truth/*.csv` ensures files in this directory are tracked.
